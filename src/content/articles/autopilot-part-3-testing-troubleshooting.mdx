## Deploy Your First Policy

Now that your device is enrolled, let's verify you can actually manage it by deploying a simple policy.

### Example: Rename the Computer Automatically

Let's create a configuration profile that renames the enrolled computer.

1. In Endpoint Manager, go to **Devices â†’ Configuration profiles**
2. Click **Create profile**
3. Select **Windows 10 and later**
4. Choose **Settings catalog** (or use a template)

> **Screenshot:** *Create configuration profile*

For this example, let's use the **Device rename** template:

1. Search for "device rename" in settings
2. Configure the rename rule:
   - **Name pattern:** `{first}-{serial}` (e.g., "zak-DELL1234")
   - **Action:** Update

3. Assign to your test device or group

> **Screenshot:** *Configure device rename policy*

4. Click **Create**

### Trigger a Sync

After creating the policy, you need to push it to the device:

## Common Issues & How to Fix Them

Don't panic if things don't work perfectly the first time. Here are the most common Autopilot issues and their solutions.

### Issue: "Profile Not Found" Error

**What you see:** During OOBE, the device says it can't find an Autopilot profile.

**Cause:** The profile hasn't been assigned to the device or group.

**Fix:**
1. Go to **Devices â†’ Windows â†’ Autopilot deployment profiles**
2. Open your profile
3. Click **Assign**
4. Make sure the device or its group is selected

> **Screenshot:** *Profile assignment error*

### Issue: Enrollment Failed

**What you see:** "Enrollment failed" or similar error during setup.

**Cause:** Usually a license issue â€” the user doesn't have an Intune license.

**Fix:**
1. Check the user's license in **Microsoft 365 Admin Center**
2. Ensure they have Intune or M365 Business Premium
3. Assign the license if missing

### Issue: Can't Join Azure AD

**What you see:** Device can't connect to Azure AD during enrollment.

**Cause:** Network issues or Azure AD settings.

**Fix:**
1. Verify the device has internet access
2. Check **Devices â†’ Azure AD devices â†’ Device settings** in Endpoint Manager
3. Ensure "Users may join devices to Azure AD" is enabled
4. Check that the user isn't hitting device limits

### Issue: Device Not Appearing After Import

**What you see:** You imported the CSV but the device doesn't show up anywhere.

**Cause:** The hardware hash might be incorrect or the import failed silently.

**Fix:**
1. Go to **Devices â†’ Windows â†’ Windows devices**
2. Check for import errors
3. Re-run the PowerShell script and re-import
4. Make sure the CSV format is correct (no extra headers or columns)

### Issue: Policy Not Applying

**What you see:** You created a policy but it hasn't reached the device.

**Cause:** Timing or assignment issues.

**Fix:**
1. Check **Device status** in the policy details
2. Verify the device is in the assigned group
3. Click **Sync** on the device
4. Wait â€” it can take up to 24 hours but usually faster

> ðŸ’¡ **Tip:** Most issues resolve by having the device **sync** or by **restarting** the device.
